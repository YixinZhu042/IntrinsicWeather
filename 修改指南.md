# 将 nerfies.github.io 改为自己项目页面的修改指南

本文档列出 `index.html` 中需要修改的内容，按从上到下顺序对应页面结构。

---

## 1. 头部 meta 与标题（约第 5–9 行）

| 位置 | 原内容 | 修改为 |
|------|--------|--------|
| `<meta name="description">` | Nerfies 相关描述 | 你的项目一句话描述（用于搜索引擎） |
| `<meta name="keywords">` | Nerfies, D-NeRF, NeRF | 你的项目关键词，逗号分隔 |
| `<title>` | Nerfies: Deformable Neural Radiance Fields | **你的项目名称**（浏览器标签标题） |

---

## 2. Google Analytics（约第 11–22 行）

- **若不需要统计**：整段删除（从 `<!-- Global site tag -->` 到 `</script>`）。
- **若使用自己的 GA**：把 `G-PYVRSFMDRL` 换成你的 Measurement ID。

---

## 3. 导航栏 Navbar（约第 44–82 行）

| 位置 | 修改说明 |
|------|----------|
| 第 54 行 `href="https://keunhong.com"` | 改成你的个人/实验室首页，或删除该链接 |
| 第 59–77 行「More Research」下拉 | 改成你的其他项目链接，或整块删除 |
| 若不需要导航栏 | 可删除整个 `<nav>...</nav>` 块（第 44–82 行） |

---

## 4. Hero 主标题区（约第 84–181 行）

### 4.1 标题与作者（约第 89–116 行）

| 位置 | 原内容 | 修改为 |
|------|--------|--------|
| 第 89 行 `publication-title` | Nerfies: Deformable Neural Radiance Fields | **你的论文/项目标题** |
| 第 91–110 行 | 七位作者及链接 | 改成你的作者列表：`<a href="个人主页">姓名</a><sup>机构编号</sup>`，用逗号分隔 |
| 第 114–115 行 | University of Washington, Google Research | **你的机构名称**（与上标编号对应） |

### 4.2 链接按钮（约第 119–173 行）

按需修改或删除，保留需要的即可：

| 按钮 | 行号 | 修改 |
|------|------|------|
| Paper | 122–128 | `href` 改为你的 PDF 链接（如 arXiv 或会议 PDF） |
| arXiv | 130–137 | `href` 改为你的 arXiv 链接，若无则删整段 |
| Video | 140–149 | `href` 改为你的视频链接（如 YouTube），若无则删整段 |
| Code | 152–161 | `href` 改为你的 GitHub 仓库链接 |
| Data | 164–172 | 若有数据集则改链接，若无则删整段 |

---

## 5. Teaser  teaser 视频区（约第 183–198 行）

| 位置 | 修改说明 |
|------|----------|
| 第 185–187 行 `<source src="...">` | 换成你的 teaser 视频路径（如 `./static/videos/你的teaser.mp4`） |
| 第 190–193 行 `subtitle` | 改成一句项目 slogan/副标题（如「XXX 实现了……」） |

**注意**：需在 `static/videos/` 下放置对应视频文件。

---

## 6. 结果轮播 Carousel（约第 201–255 行）

- 每个 `<div class="item item-xxx">` 里是一个视频。
- 按你的结果数量增删 `item`：
  - **减少**：删掉不需要的整块 `<div class="item">...</div>`。
  - **增加**：复制一个 `item`，改 `id`、`class` 里的名字和 `<source src="...">` 路径。
- 所有视频路径都指向 `./static/videos/`，请把实际视频放到该目录并保持文件名一致。

---

## 7. Abstract 摘要（约第 261–294 行）

- 第 264 行：可把「Abstract」改成「摘要」或保留英文。
- 第 266–291 行：把三段话全部替换为**你的论文摘要**（可保留多段 `<p>...</p>`）。

---

## 8. 论文视频 Video（约第 297–307 行）

| 位置 | 修改说明 |
|------|----------|
| 第 303 行 `iframe src="..."` | 换成你的 YouTube 嵌入链接。格式：`https://www.youtube.com/embed/你的视频ID` |

若无论文视频，可整段删除该 `<div class="columns">...</div>`。

---

## 9. 各小节内容（约第 313–405 行）

以下每一块都可以整段保留、改内容或删除：

| 小节 | 约行号 | 修改说明 |
|------|--------|----------|
| **Visual Effects** | 317–331 | 标题 + 说明文字 + 视频；换成你的效果说明和 `./static/videos/` 下视频 |
| **Matting** | 334–350 | 同上，改为你的方法展示或删除 |
| **Animation - Interpolating** | 355–384 | 插值演示：替换说明文字、`./static/images/` 下首尾帧图片、以及 JS 里用的资源（若需插值需配合 `static/js/index.js`） |
| **Animation - Re-rendering** | 387–406 | 替换说明和 `./static/videos/replay.mp4`，或删除 |

不需要的小节直接删掉整个对应 `<div class="column">...</div>` 或 `<div class="columns">...</div>`。

---

## 10. Related Links 相关链接（约第 409–429 行）

- 第 411 行：可改为「Related Work」或「相关链接」等。
- 第 414–426 行：替换成你论文里引用的相关项目链接和简短描述；若不需要可整段删除。

## 11. BibTeX（约第 434–444 行）

| 位置 | 修改说明 |
|------|----------|
| 第 436–441 行 `<pre><code>...</code></pre>` | 换成你的 BibTeX（作者、title、journal、year 等） |

## 12. 页脚 Footer（约第 448–473 行）

| 位置 | 修改说明 |
|------|----------|
| 第 451 行 `href="./static/videos/nerfies_paper.pdf"` | 改为你的 PDF 路径，或删除该链接 |
| 第 454 行 `href="https://github.com/keunhong"` | 改为你的 GitHub 主页或仓库 |
| 第 463–465 行 | 若保留 CC 协议可不动，否则改成你的版权/许可说明 |
| 第 468 行 `href="https://github.com/nerfies/nerfies.github.io"` | 改为你当前项目页的 GitHub 地址（若你 fork 并改成了自己的仓库） |
| 第 469–471 行 | 说明可改为「本页基于 XXX 模板」或删除 |

---

## 13. 静态资源（不直接写在 index 里，但必须准备）

| 类型 | 路径 | 说明 |
|------|------|------|
| 视频 | `static/videos/` | teaser、轮播、各小节演示视频，格式建议 mp4 |
| 图片 | `static/images/` | 如插值首尾帧、favicon 等 |
| 图标 | `static/images/favicon.svg` | 浏览器标签图标，可替换成自己的 |

---

## 14. 可选：精简页面结构

若你希望页面更简洁，可以：

1. **删除整块**：导航栏、轮播、Visual Effects、Matting、Animation、Related Links 中不需要的整块 section。
2. **保留最小集合**：标题 + 作者 + 链接按钮 + Teaser + Abstract + Video + BibTeX + Footer。
3. 删除某块后，若 `static/js/index.js` 里有对应 id（如轮播、插值滑块），可一并删除或注释掉相关 JS，避免报错。

- [ ] Abstract 全文
- [ ] 论文 YouTube 嵌入链接
- [ ] 各小节标题、文字与视频/图片
- [ ] Related Links 或删除
- [ ] BibTeX
- [ ] Footer 链接与说明

按上述项逐项修改后，`index.html` 即可作为你自己的项目页面使用。部署到 GitHub Pages 时，确保仓库里包含 `index.html` 和 `static/` 下的资源即可。
